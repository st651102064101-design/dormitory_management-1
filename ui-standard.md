นี่คือสรุปค่าพารามิเตอร์ต่างๆ ตามมาตรฐาน Apple Human Interface Guidelines (HIG) สำหรับการนำไปเขียนโค้ด (CSS, Swift, SwiftUI หรือเครื่องมือ Design อื่นๆ) สำหรับสร้าง Alert และ Button ครับ

### 1. ค่าสี (HEX Colors)
สีของ Apple จะมีการปรับเปลี่ยน (Dynamic) ตาม Light Mode และ Dark Mode เพื่อความสบายตาและ Contrast ที่เหมาะสม:

| ตัวแปรสี (Color) | Light Mode | Dark Mode | การนำไปใช้งาน |
| :--- | :--- | :--- | :--- |
| **System Blue** | `#007AFF` | `#0A84FF` | สีหลัก (Primary/Accent) สำหรับข้อความลิงก์ ปุ่มปกติ หรือปุ่ม OK ใน Alert, |
| **System Red** | `#FF3B30` | `#FF453A` | สำหรับปุ่มที่มีผลร้ายแรง (Destructive Action) เช่น ปุ่มลบเนื้อหา (Delete),, |
| **Background (Alert)** | `N/A (Blur)` | `N/A (Blur)` | Alert ของ iOS จะไม่มีการระบุ HEX สีทึบ แต่จะใช้พื้นหลังเป็น **Visual Effects View (Translucent / Blur)** พร้อมกับ Backdrop Filter เพื่อดึงสีจากพื้นหลังมาเบลอ, |
| **Background (Solid)** | `#FFFFFF` | `#1E1E1E` หรือ `#000000` | หากต้องใช้สีทึบแทนการเบลอ ค่า Control Background ใน Light Mode จะเป็นสีขาว และ Dark mode มักใช้สีเทาเข้มถึงดำ, |

---

### 2. ค่าความโค้งมน (Border Radius)
Apple ไม่ได้มีค่าตายตัวค่าเดียวเสมอไป แต่จะปรับไปตามความเหมาะสมของขนาด UI Components ดังนี้:

| คอมโพเนนต์ (Component) | ค่า Border Radius | รายละเอียดอ้างอิง |
| :--- | :--- | :--- |
| **iOS Alert ทั่วไป** | `10px` ถึง `13px` | ไม่มีการกำหนดตัวเลขใน HIG ชัดเจน แต่เพื่อให้ได้ความรู้สึก Native แบบ iOS นักออกแบบมักใช้ค่าความโค้งที่ **10px, 12px หรือ 13px** |
| **visionOS Alert** | `16pt` | สำหรับ Accessory view ในหน้าต่าง Alert บนระบบ visionOS จะใช้ความโค้งที่ 16pt |
| **Button (ปุ่มกด)** | `System-managed` | ความโค้งของปุ่มระบบจะจัดการให้ตามขนาดปุ่มโดยอัตโนมัติ (Small, Medium, Large) |
| **ปุ่มจำเพาะ (เช่น Apple Pay)** | `4.0pt` | ค่า Default ของความโค้งปุ่มระบบ เช่น PKPaymentButton ถูกตั้งค่าเริ่มต้นไว้ที่ 4.0 |

### 3. ตัวอักษร (Typography)
อ้างอิงจากฟอนต์ **San Francisco (SF Pro Text)** ซึ่งเป็นฟอนต์ระบบสำหรับข้อความที่มีขนาด 19pt ลงมา Apple ไม่ได้เน้นแค่ขนาด แต่ให้ความสำคัญกับ "น้ำหนัก" (Weight) ของฟอนต์ในการแยกแยะข้อมูล:

| การใช้งานใน Alert / Button | ขนาด (Font Size) | น้ำหนัก (Weight) | ระยะห่าง (Tracking) | ข้อแนะนำ |
| :--- | :--- | :--- | :--- | :--- |
| **หัวข้อ Alert (Title)** | `17pt` | **Semi-bold / Bold** | `-0.43px` | ใช้ขนาดเท่า Text มาตรฐาน แต่ใช้ **น้ำหนักฟอนต์ที่หนาขึ้น** เพื่อให้ดูโดดเด่นและประหยัดพื้นที่, |
| **เนื้อหา Alert (Message)** | `13pt` | Regular | `-0.08px` | เพื่อไม่ให้กินพื้นที่มากเกินไป ใช้ฟอนต์ขนาดเล็กลงในการอธิบายเพิ่มเติม, |
| **Button Text (ปุ่มทั่วไป)** | `17pt` | **Medium** | `-0.43px` | ข้อความปุ่มมาตรฐานของ iOS จะอยู่ที่ 17pt |
| **Alert Button: ปุ่มหลัก (Default)** | `17pt` | **Medium / Bold** | `-0.43px` | ปุ่มที่ผู้ใช้ควรจะกดเพื่อตกลง (เช่น OK) จะใช้ความหนาเข้าช่วยดึงดูดสายตา |
| **Alert Button: ปุ่มทางเลือก (Cancel)** | `17pt` | Regular | `-0.43px` | ปุ่มยกเลิกจะอยู่ที่ด้านซ้ายเสมอ และใช้ขนาด/น้ำหนักแบบปกติ |

**ข้อควรระวังในการเขียนโค้ด:**
*   **พื้นที่กดปุ่ม (Minimum Touch Target):** แม้ตัวอักษรจะเล็ก แต่พื้นที่สำหรับกดปุ่มทุกชนิด **ต้องไม่ต่ำกว่า 44x44 pt** เสมอ เพื่อให้ใช้นิ้วทัชได้ง่าย,
*   **การวางปุ่ม (Button Placement):** สำหรับ Alert ที่มี 2 ปุ่ม ปุ่มกดยกเลิก (Cancel) ควรอยู่ด้านซ้ายเสมอ ส่วนปุ่มยืนยันหรือปุ่มหลัก (Default/Action) ให้วางไว้ด้านขวา,
